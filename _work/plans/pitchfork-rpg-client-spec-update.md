# 1. Migration Goal

The objective is not to shorten or simplify the RPG specification.

The objective is to transform the RPG from:

```text
Standalone Life-Powered RPG
```

into:

```text
Node-Aware Symbolic Client
for the Pancakes / Pitchfork Ecosystem
```

while preserving:

* gameplay depth,
* crafting systems,
* gathering systems,
* lore,
* mentors,
* archetypes,
* attunements,
* sanctuary progression,
* economic symbolism,
* educational intent.

---

# 2. Core Architectural Shift

## Original Assumption

The original document implicitly treats the RPG as the primary system.

```text
Life Activity
→ RPG
→ Progress
```

## New Assumption

The RPG becomes one client among many.

```text
Life Activity
→ Pancakes Client
→ Pitchfork Event
→ Settlement
→ Projection
→ RPG Interpretation
```

The RPG no longer owns life tracking.

The RPG owns symbolic interpretation.

# 7. Educational Objectives

The RPG should explicitly teach:

* value creation,
* stewardship,
* exchange,
* contracts,
* cooperation,
* commons management,
* governance,
* accounting,
* resource allocation.

Recognition of their real-world counterparts should emerge gradually.

The Pitchfork RPG is no longer merely a game.

```text

for humane life computing,
```
