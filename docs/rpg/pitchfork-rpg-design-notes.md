# Design Description: A Metaphysical Life-Economy RPG

## 1. Core Premise

This is a game ecosystem where real-world human activity powers a mythic, magical civilization.

The player is not simply managing habits. The player is synchronizing their real life with a symbolic world. Walking, sleeping, eating, studying, journaling, caregiving, exercising, organizing, resting, and creating all become forms of magical participation.

The system is not just:

> “Do tasks, get XP.”

It is:

> “Live with rhythm, intention, and care, and your presence shapes a shared magical reality.”

The ultimate ambition is to recognize forms of value that modern economies often ignore: household management, caregiving, emotional labor, health maintenance, learning, consistency, stewardship, creativity, and community support.

The first product can be much smaller: a single-player browser RPG inspired by *Kingdom of Loathing* and *Habitica*, eventually expanding into a multi-client, MMO-like ecosystem.

---

# 2. The Three Worlds

The system is built around three connected worlds.

## A. The Real World

This is the player’s actual life.

Real-world actions include:

* walking
* sleeping
* exercising
* eating
* cooking
* cleaning
* studying
* doing homework
* journaling
* meditating
* caregiving
* organizing a household
* managing cycles
* resting
* creating
* maintaining relationships

These actions are not treated as mere productivity tasks. They are treated as forms of life-energy, ritual participation, and stewardship.

The system should avoid implying that people must optimize every minute. It should reward rhythm, presence, care, consistency, and alignment.

---

## B. The Magic World

This is the experiential RPG layer.

The player sees:

* magic
* rituals
* crafting
* gathering
* exploration
* guilds
* covenants
* relics
* resources
* sanctuaries
* spell systems
* attunements
* archetypes
* magical ecology

The Magic World should feel mythic, atmospheric, and alive.

The player should not feel like:

> “My Fitbit synced and gave me +8 XP.”

They should feel like:

> “My morning walk gathered ember moss from the ley road.”

or:

> “My sleep restored lucid dew in the dream garden.”

or:

> “My household rituals reduced entropy in my sanctuary.”

The Magic World is the symbolic interpretation of lived reality.

---

## C. The Commerce World

This is the economic, contractual, and coordination layer.

It contains the concepts analogous to crypto and smart contracts, but expressed in fantasy-native language.

Possible mappings:

| Technical Concept | In-World Concept                    |
| ----------------- | ----------------------------------- |
| Wallet            | Soul sigil                          |
| Private key       | True name                           |
| Smart contract    | Binding covenant                    |
| Token             | Essence, shard, sigil, relic        |
| DAO               | Mage council, guild, covenant order |
| Staking           | Mana attunement                     |
| Escrow            | Sealed ritual                       |
| Validator         | Rune keeper                         |
| Consensus         | Ritual synchronization              |
| Marketplace       | Bazaar, trade hall, astral exchange |
| NFT               | Soulbound relic or crafted artifact |
| Reputation        | Renown, trust, lineage              |

The Commerce World does not need to begin with real blockchain infrastructure. Early on, it can be simulated internally. The important thing is the logic: ownership, trust, contracts, exchange, reputation, and shared contribution.

# 3. The Central Design Principle

The system is not a productivity app with fantasy decoration.

It is an RPG powered by reality.

That distinction is critical.

A weak version says:

> “Do chores to earn points.”

A strong version says:

> “Your life patterns generate magical resources, shape your identity, and contribute to a civilization.”

The core loop:

```text
Real-world action
→ symbolic interpretation
→ magical resource generation
→ gathering / crafting / ritual use
→ identity progression
→ world contribution
→ renewed motivation for real-world action
```

The system should feel more like cultivation than conquest.

# 5. Gathering and Crafting as the Core Loop

The strongest gameplay direction is gathering and crafting, not combat.

This came from the insight that one of the most compelling activities in *World of Warcraft* is gathering flowers and crafting.

That is important because gathering/crafting naturally maps to real-world behavior.

Examples:

| Real-World Activity      | Magical Resource                       |
| ------------------------ | -------------------------------------- |
| Walking                  | Ember moss, road dust, kinetic essence |
| Sleep                    | Lucid dew, dream resin                 |
| Journaling               | Memory ink, reflection glass           |
| Meditation               | Void crystal, stillwater               |
| Exercise                 | Vitality ore, fire root                |
| Cooking                  | Nourishment essence, hearth spice      |
| Cleaning                 | Order salt, entropy ash                |
| Studying                 | Rune fragments, scholar’s dust         |
| Caregiving               | Hearthlight, sanctuary thread          |
| Menstrual cycle tracking | Moonwater, crimson bloom, cycle sigils |
| Household planning       | Temporal thread, steward’s wax         |
| Budgeting                | Ledger stone, trade glyphs             |

The player then uses these materials to:

* brew potions
* craft relics
* build sanctuaries
* fulfill rituals
* strengthen guilds
* trade with others
* unlock story paths
* stabilize regions of the world
* create magical infrastructure

This makes self-improvement feel like worldbuilding.

# 6. Magic System

Magic is powered by resonance.

## Resonance

Resonance is the degree of alignment between:

* the player’s habits
* their chosen archetype
* their rituals
* their in-game identity
* their social role
* their contributions to the world

This avoids reducing the game to raw metrics.

Instead of:

> “10,000 steps = 100 fire damage”

the system says:

> “Your recent movement patterns have strengthened your Fire and Road resonance.”

Resonance should be interpretive, not mechanical in an obvious one-to-one way.

---

## Attunements

Players develop attunements based on life patterns.

Examples:

| Attunement | Associated Life Patterns                 |
| ---------- | ---------------------------------------- |
| Fire       | exercise, intensity, courage, action     |
| Dream      | sleep, imagination, reflection           |
| Stone      | routine, consistency, household order    |
| Water      | emotional care, recovery, nourishment    |
| Air        | study, curiosity, writing, communication |
| Moon       | cycles, rest, fertility, reflection      |
| Root       | caregiving, gardening, food, home        |
| Void       | meditation, silence, focus               |
| Gold       | commerce, budgeting, trade               |
| Thread     | scheduling, coordination, planning       |

These attunements affect:

* available recipes
* gathering outcomes
* ritual compatibility
* archetype evolution
* faction alignment
* crafted item properties
* world influence

---

## Rituals

Rituals are structured actions in the Magic World.

They may require:

* resources
* timing
* attunement
* consistency
* collaboration
* a covenant
* a location
* a lunar phase
* a crafted object

Examples:

* Brew a Sleep Draught from lucid dew.
* Bind a household sanctuary with hearthlight.
* Stabilize a leyline through seven days of walking.
* Create a Study Sigil from three completed homework sessions.
* Join a guild ritual during the full moon.
* Craft a Caretaker’s Lantern from caregiving actions.

Rituals are where magic, habits, crafting, and economics meet.

# 10. Economic Philosophy

The economy should not directly become:

> “Do more life tasks, earn more money.”

That would risk becoming exploitative, especially for vulnerable users.

Instead, the better model is a capped participation economy.

---

## Participation-Based Rewards

Users can earn a limited number of tokens per time cycle.

For example:

* one fire token per day
* three total essence tokens per day
* weekly ritual rewards
* lunar cycle rewards
* seasonal harvests

The cap is crucial.

Without caps:

```text
more optimization → more income
```

With caps:

```text
basic participation → baseline reward access
```

That shifts the system away from gig labor and toward symbolic basic income.

---

## Verification Philosophy

The system does not need to police sincerity aggressively.

If someone writes a fake journal entry but engages with the system in the shape of a regular participant, the platform can still treat that as valid participation.

This is a major philosophical choice.

The platform is not trying to become surveillance infrastructure.

It is trying to create consensual ritual structures.

The important question is not always:

> “Was this action objectively verified?”

Sometimes the better question is:

> “Did this person show up and participate in the symbolic order?”

This avoids turning the system into a biometric productivity regime.

There can be multiple economic layers.

### 1. Basic Essence Tokens

Capped, participation-based rewards.

These might represent:

* fire essence
* dream essence
* hearth essence

* study essence

They provide baseline participation value.

Players combine resources into higher-order objects.

Examples:

* potions
* relics
* scrolls
* sanctuary items
* ritual tools
* guild infrastructure
* magical contracts
* decorative artifacts

Crafted items can be tradable.

This is healthier than selling raw behavioral data because value comes from symbolic creation, not pure task extraction.

### 3. Soulbound Achievements

Some achievements should not be tradable.

Examples:

* personal milestones
* archetype progression
* reputation
* covenant trust
* major rituals completed
* recovery journeys
* household stewardship arcs

These preserve meaning and prevent total financialization of identity.

---

### 4. Cooperative Pools

Guilds, covenants, households, or communities can pool resources.

Possible uses:

* mutual aid
* emergency funds
* escape grants
* childcare support
* household support
* education support
* ritual infrastructure
* community projects

This is where the platform can support vulnerable people without reducing them to extractable labor units.

# 11. The “Pay People for Household Work” Mission

The deeper mission is to recognize and reward stewardship.

The risky framing is:

> “Women get paid to run households.”

The stronger framing is:

> “People who perform essential stewardship work can receive recognition, status, resources, and economic support.”

This includes:

* mothers
* fathers
* caregivers
* homemakers
* disabled people
* elders
* community organizers
* older siblings
* mutual aid workers
* teachers
* household managers

The platform can still specifically help women, especially because women disproportionately perform unpaid household and caregiving labor. But the product framing should avoid locking people into gender roles.

# 12. Safety and Ethical Boundaries

The system must avoid becoming:

* surveillance capitalism
* gamified domestic extraction
* productivity coercion
* behavioral wage labor
* health-data speculation
* manipulative streak addiction
* exploitative token farming
* gender-role reinforcement

Key safeguards:

## A. Hard Reward Caps

No infinite farming of real-world behavior.

## B. Privacy by Default

Sensitive data should be private unless explicitly shared.

Especially:

* menstrual data
* health data
* location data
* household data
* caregiving patterns
* emotional logs

## C. Symbolic, Not Forensic

The system should not require invasive proof for every action.

## D. Soulbound Identity

Not everything should be tradable.

## E. Anti-Coercion Design

Users should be able to:

* hide activity
* pause participation
* opt out of guild pressure
* avoid public streaks
* disengage without punishment
* keep economic and personal data separate

## F. Low-Stakes Baseline

The basic income-like component should not require extreme performance.

## G. Multiple Valid Paths

The system should not imply that only fitness, productivity, motherhood, or discipline are valuable.

Rest, recovery, care, learning, and maintenance should all matter.

# 14. The First Product Definition

The first product should probably be:

> A single-player, text-heavy, browser-based life-powered gathering and crafting RPG.

Not:

> a full MMO
> a crypto platform
> a quantified-self dashboard
> a domestic labor marketplace

The MVP should focus on the emotional loop.

A possible first-session experience:

1. User creates a soul sigil.
2. User chooses an archetype.
3. User enters a small magical region.
4. User selects daily rituals.
5. User logs or syncs real-world actions.
6. The system interprets them symbolically.
7. The user gathers resources.
8. The user crafts something.
9. The crafted item changes their sanctuary or unlocks a path.
10. The user feels their real life has become mythically meaningful.

That is the foundation.

# 15. The Big Vision

At full scale, this becomes:

> A shared magical civilization powered by human life patterns, where care, discipline, creativity, study, health, household stewardship, and community support become visible, valuable, and economically meaningful.

It is part RPG, part habit system, part symbolic economy, part mutual-aid network, part life interface, part MMO.

The core innovation is not crypto.

The core innovation is:

> turning invisible forms of human value into magical, social, and eventually economic presence.

The best version of this system does not say:

> “Optimize yourself.”

It says:

> “Your way of caring for life matters. Bring it into the world.”
